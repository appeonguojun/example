1. Introduction

1.1 Welcome to Appeon PowerBuilder 2017 R2

Welcome to Appeon PowerBuilder!

Appeon PowerBuilder is an enterprise development tool that allows you to build data-driven business applications and components. It is one of a suite of Appeon products that together provide the tools to develop client/server, Web, and mobile applications.

1.2 Components of PowerBuilder

Different PowerBuilder 2017 R2 editions consist of different components. The PowerBuilder 2017 R2 Universal Edition contains both the PowerBuilder IDE and the PowerServer (PB Edition) products.

Component Universal Edition Standard/Cloud Edition
PowerBuilder IDE Yes Yes

PowerServer (PB Edition)

  • PowerServer Mobile (.NET)
  • PowerServer Toolkit
  • PowerServer Help
  • PowerServer Web Component
 Yes No
  • PowerBuilder IDE - The development platform for rapidly building distributed, Web, mobile, and client/server applications.
  • PowerServer (PB Edition) - Provides mobile capabilities for the PowerBuilder applications.
    • PowerServer Mobile (.NET) - A set of server components that are installed to the Microsoft IIS application server and provide the mobile apps with runtime and management services such as data connectivity, DataWindow support, transaction management, and security.
    • PowerServer Toolkit - A set of tools that are installed to the PowerBuilder IDE and deploy PowerBuilder apps to the PowerServer as Web or mobile apps.
    • PowerServer Help - A compiled HTML Help that provides easy use to all user documents.
    • PowerServer Web Component - A series of C++ and JavaScript libraries that are installed to Apache or Microsoft IIS Web server. It is only necessary when configuring a cluster of PowerServers.

Different PowerBuilder 2017 R2 editions contain different features.

2. System Requirements

Before installing PowerBuilder 2017 R2 Universal Edition, please ensure that your system meets the requirements below:

For installing PowerBuilder & PowerServer Toolkit

  • Windows 10, Windows 8.1, or Windows 7
    Windows Server 2016/2012/2008 (supports application runtime only, not development)
  • (Optional) SAP SQL Anywhere 17.0, 16.0, or 12.0 (for running the PowerBuilder demos)

For installing PowerServer Mobile (.NET)

  • (Recommended) Windows Server 2016, 2012 R2, or 2008 R2
    (Optional) Windows 10/8.1/7
  • Microsoft IIS 10, 8.0, or 7.5
  • Microsoft .NET Framework 4.x
    For more information about the system requirements of PowerServer, check the PowerServer Readme file and the PowerServer Installation Guide.

3. Installing PowerBuilder 2017 R2

The PowerBuilder Universal Edition installation package can install both the PowerBuilder IDE and the PowerServer (PB Edition) products.

Note that:
1) If an older version of PowerBuilder IDE (such as 2017) is installed, uninstall the old version first and then install 2017 R2.
2) Before selecting to install PowerServer (PB Edition), please read the PowerServer Readme and PowerServer Settings files carefully first.

Step 1: Double click Autorun.exe in the universal edition installation package.

Step 2: Click the Install button on the start page.

Step 3: Click Next to proceed.

Step 4: On the License Agreement page, accept the terms in the license agreement page and click Next.

Step 5: On the Customer Information page, enter your name and your company’s name and click Next.

Step 6: On the Select Products page, select PowerBuilder IDE, PowerServer Mobile, PowerServer Toolkit, and PowerServer Help, and click Next to continue.

The PowerBuilder IDE installation starts first.

  1. On the Choose Destination Location page, click Next to accept the destination path shown, or click Browse and select a different path, click OK, and click Next.
  2. On the Choose Destination Location for Shared Files page, click Next to accept the destination path shown, or click Browse and select a different path, click OK, and click Next. Use the same folder for shared files for all products in this set.
  3. On the Specify the SQL Anywhere Engine page, select Yes and then specify the location of a SQL Anywhere engine that will be used to run the demo database (PB demo DB) or select No if no SQL Anywhere engine, and click Next.
  4. In the Select Components page, unselect any components you do not want to install, and click Next.
  5. On the Select Program Folder page, select the program folder to which program icons will be added, or specify a new one; then click Next.
  6. On the Start Copying Files page, review your settings, and then click Next to begin installing files for PowerBuilder.
    7.

The PowerServer Mobile installation starts automatically when the PowerBuilder installation is completed.

  1. Click Next until you reach the screen for specifying the IIS Web site where PowerServer Mobile will be installed. Make sure Select an existing Web Site and Default Web Site are selected.
  2. Click Next. Choose whether to install the Demo Applications. If you have SQL Anywhere database server installed, select to install the demo; otherwise, do not install the demo since the demo requires SQL Anywhere database server to be previously installed.
  3. If you selected to install the Demo Applications, specify the path for the SQL Anywhere database server engine.
  4. Click Next until Setup begins installing files for PowerServer Mobile.
    5.

The PowerServer Toolkit installation starts automatically when the PowerServer Mobile installation is completed. Click Next until Setup begins installing files for PowerServer Toolkit.

The PowerServer Help installation starts automatically when the PowerServer Toolkit installation is completed. Click Next until Setup begins installing files for PowerServer Help.

Step 10: Click Finish when the PowerServer Mobile, PowerServer Toolkit, and PowerServer Help installations are completed.

Step 11: After closing all the windows during installation, the System Reboot Required screen is displayed. Select the Yes, I want to restart my computer now checkbox to reboot the system and then click Finish.

4. Activating PowerBuilder 2017 R2

You are required to activate PowerBuilder and PowerServer first before using it. If you do not have a valid account, please contact your license administrator to create an account for you. And depending on whether the machine where PowerBuilder/PowerServer is installed has Internet access or not, the account should be assigned with an online license or offline license by your license administrator.

Note that PowerBuilder and PowerServer are activated separately. Follow the "Activating PowerBuilder" section in the Appeon License User Guide to activate PowerBuilder, and follow the "Activating PowerServer Mobile (PB Edition)" section in the Appeon License User Guide to activate PowerServer (PB Edition).

5. Uninstalling PowerBuilder 2017 R2

5.1 Uninstalling PowerBuilder

Before you uninstall PowerBuilder, make sure you have logged out PowerBuilder if using an online license, or have deactivated the offline license. For how to log out PowerBuilder or deactivate an offline license, refer to Logging out or De-activating an offline license in the Appeon License User Guide.

Launch Uninstall a program in the Windows Control Panel. Select Appeon PowerBuilder 2017 R2 and click the Uninstall button to begin the uninstall process.

5.2 Uninstalling PowerServer (PB Edition)

Follow instructions in Deactivating an online license and Deactivating an offline license in the Appeon License User Guide to de-activate PowerServer (PB Edition) and then take the same steps as above to uninstall PowerServer Mobile, PowerServer Toolkit, and PowerServer Help.
